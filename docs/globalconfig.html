Global Configuration

  
Some of the global computation parameters can be defined with the help of the CharacterBuilder System menu. It provides a dialog window for defining the Allowed Dialog Acts that are used for the elicitation of emotions. Also, there is a dialog window for specifying the emotion types that should be considers for the affect computation.

Other global computation parameters should be specified directly in the relevant AML configuration file (e.g. conf/AffectComputation.aml).

Allowed Dialog Acts

For the definition of Dialog Act appraisal rules for a character, ALMA has to know, which Dialog Acts are allowed. For internal use, there is a documentation area for each Dialog Act.



Changes are instantly available for the character configuration.
Important: The character config dialog has to be re-opened! To keep the changes permanently, save them via the File-Save-Computation Parameters menu.

Available Emotions

This dialog allows the activation of emotion types that should be considered in the affect computation. Activate or deactivate the use of a specific emotion by (de-)selecting its use-checkbox. For internal use, there is a documentation area for each emotion.



Changes are available after restarting ALMA.
Important: To keep the changes permanently, save them via the File-Save-Computation Parameters menu.
